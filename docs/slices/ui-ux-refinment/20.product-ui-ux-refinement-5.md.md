Emergency fix: Standardize all dialogs/popups so actions work everywhere.

Current issue:
- New Permit submit was fixed because the confirmation dialog existed but was invisible due to `.dialog-backdrop` requiring `.open`.
- The same problem now happens for HSE approve/reject.
- Likely other pages using old dialog/popup patterns may also be broken.

This is release-blocking.

Goal:
Audit and fix all confirmation dialogs, popups, and action overlays across the app so no button silently does nothing.

Inspect all pages/components that use:
- dialog-backdrop
- modal
- confirmation
- approve
- reject
- submit
- logout
- print if relevant
- snackbar/toast/action feedback

Required pages/actions to verify:
1. New Permit
   - Submit for HSE Approval
2. HSE approvals
   - Approve submitted permit
   - Reject submitted permit with reason
3. Construction Manager approvals
   - Approve HSE-approved permit
   - Reject HSE-approved permit with reason
4. Permit details page
   - HSE approve/reject actions if available there
   - CM approve/reject actions if available there
   - Print / Save PDF
5. Logout/account dropdown if it uses confirmation
6. Any admin create/edit/deactivate confirmation if present

Required behavior:
- Every popup/dialog must open visibly.
- Every popup/dialog must close correctly.
- Confirm button must call the actual existing business action.
- Cancel button must close the dialog.
- Clicking outside should close only when appropriate.
- Escape should close when appropriate.
- While action is submitting, confirm button should be disabled.
- If action fails, show a visible error.
- If validation fails, show visible validation.
- No action should silently do nothing.

Implementation guidance:
- Prefer one consistent dialog pattern across the app.
- If the current CSS uses `.dialog-backdrop.open`, every dialog must bind `[class.open]="someSignalOrBoolean"`.
- Remove or update old `@if` dialog patterns that conflict with the new CSS.
- Do not create duplicate dialog systems.
- Keep animations/transitions working.
- Keep dark mode compatible.
- Keep mobile layout compatible.

Do not:
- Change business logic.
- Change Firestore models.
- Change Firestore rules.
- Change approval rules.
- Change permit creation rules.
- Add dependencies.
- Redesign screens.
- Refactor unrelated code.

After fixing:
- Explain every broken dialog found.
- Explain root cause.
- List changed files.
- Confirm business logic was not changed.
- Run npm run build.
