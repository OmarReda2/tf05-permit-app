The MVP functionality is complete and deployed. The current UI is cleaner than before, but it still feels like a styled CRUD app. The goal is now to refine it into a Google/Firebase-console-inspired product UI: calm, structured, professional, responsive, and highly usable.

Design direction:
- Google/Firebase-console-inspired
- Material Design inspired
- Clean enterprise SaaS
- Light surfaces
- Subtle borders
- Clear hierarchy
- Calm colors
- Strong spacing system
- Excellent desktop and mobile behavior
- No flashy dashboard
- No dark/control-room theme
- No random animations
- No emojis

Main goals:

1. Navigation system
- Rework sidebar into a clearer product navigation.
- Group related routes where appropriate.
- Permits should be treated as a parent area, with New Permit as a related child/action.
- Fix active state so /permits/new does not make both Permits and New Permit look equally active.
- Hide navigation items the current user cannot access.
- Add simple inline SVG icons or CSS icons to nav items.
- Add collapsible desktop sidebar/rail if simple and safe.
- Do not show unauthorized pages in nav.
- Keep HSE checklist management accessible without exposing full Admin.

2. Mobile navigation
- Replace the current rectangular bottom bar with a cleaner floating rounded bottom nav.
- Use equal-width tabs.
- Use only primary destinations.
- Respect role-based visibility.
- Add safe-area/bottom padding so content does not hide behind the nav.
- Do not overcrowd mobile nav.
- Admin/config pages can be accessed from secondary links, not necessarily bottom nav.

3. Header/account area
- Reorganize the header.
- Move profile/account behavior to the top-right account control.
- Show user name and role cleanly.
- Replace large Logout button with an account dropdown/menu containing Logout.
- Keep header calm and professional.
- Keep page title/content hierarchy separate from account actions.

4. Auth loading flicker
- Fix the glitch where the internal shell/sidebar/header appears briefly before redirecting to login.
- Do not show authenticated app shell until auth/profile initialization state is resolved.
- Add a simple full-page loading/splash state if needed.
- This should be a UX fix, not a business logic rewrite.

5. Design tokens
- Create or refine CSS variables for:
  - colors
  - surfaces
  - text colors
  - borders
  - spacing
  - radius
  - shadows
  - typography scale
  - status colors
- Use these consistently across the app.

6. Typography
- Improve font sizing and hierarchy for desktop and mobile.
- Use system fonts only.
- Page titles, section titles, labels, body text, table text, and badges should have consistent sizes.
- Fix browser tab title from "web" to "TF-05 Permit App".
- If practical, set dynamic page titles.

7. Dashboard
- Reduce visual noise.
- Replace overly colorful cards with calmer metric cards.
- Use minimal purposeful color.
- Add only useful analytics if simple:
  - status breakdown bar
  - approval funnel/summary
  - expiring soon list
- Do not add chart libraries.
- Do not create heavy reports.
- Recent permits should be easier to scan.

8. Permit list/table
- Make table calmer and more scannable.
- Reduce excessive badge color.
- Use badges carefully for status/risk/expiry.
- Improve column spacing and alignment.
- Improve filters: make them a compact toolbar, not a large distracting card.
- Keep separate filters for workflow status and expiry state.
- Mobile must use cards, not a wide table.

9. Permit details
- Make details feel like a structured permit dossier.
- Use clear sections:
  - summary
  - work details
  - checklist
  - approvals
  - events
- Keep print behavior intact.
- Actions should be clear but not visually noisy.

10. New permit form
- Improve form flow and readability.
- Use clear sections and helper text.
- Make validation messages calm and visible.
- Keep checklist completion easy on mobile.
- Avoid giant empty spaces and messy alignment.

11. Admin/checklist pages
- Improve form organization.
- Checklist item form currently feels messy.
- Create better layout:
  - permit type selector
  - create/edit checklist item section
  - checklist items list
- Use responsive grid on desktop and stacked layout on mobile.
- Keep HSE and Admin experiences clean.

12. Activity log
- Improve loading state.
- Use skeleton or subtle progress instead of plain placeholder box.
- Event list should feel like a clean timeline/list.
- Show action, actor, role, permit, comment, and time clearly.

13. Dark/light mode
- Add a light/dark mode toggle only if it can be done cleanly with CSS variables.
- Default to light mode.
- Store preference in localStorage.
- Dark mode must align with the app color system, not random inverted colors.
- Do not break print view.

14. Loading/empty/error states
- Standardize loading states.
- Prefer skeletons or subtle linear progress.
- Empty states must be role-aware.
- Do not show "Create your first permit" to users who cannot create permits.
- Error states should be clear and not scary.

15. Main layout spacing
- Fix page container padding/margins.
- Avoid too much unused space at the right edge.
- Use consistent responsive content width.
- Tables can use more width.
- Forms can use constrained width for readability.
- Cards should align consistently.
